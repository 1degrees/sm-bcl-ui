# 组件库概览

## 🎨 设计理念

@xfsm/bcl-ui 组件库秉持"美观、实用、高效"的设计理念，致力于为公司内部业务系统提供一致、专业的用户界面。我们融合了现代前端设计趋势与公司业务场景需求，打造了一套符合企业级应用标准的组件体系。

### 核心设计原则

- **一致性**：统一的设计语言和交互模式，确保不同业务系统的视觉协调
- **可定制性**：灵活的主题系统，支持业务差异化需求
- **易用性**：简洁直观的API设计，降低开发成本
- **高性能**：优化的组件实现，确保复杂业务场景下的流畅体验

## 🎯 已实现组件

### AudioPlayer 音频播放器

现代化的音频播放组件，支持播放控制、进度调节、音量控制等核心功能。

**核心特性：**
- 支持多种音频格式
- 响应式设计，适配不同设备
- 流畅的动画效果
- 完整的TypeScript类型支持
- 支持自定义主题

**应用场景：**
- 音频内容展示
- 语音消息播放
- 音乐播放器
- 播客应用

```
├── types/              # 类型定义
└── index.ts            # 组件库入口
```

## 🏢 公司内部开发环境集成

### 开发流程

1. **组件开发**：在 `src/components` 目录下创建组件
2. **本地测试**：使用 `pnpm dev` 启动开发服务器进行预览
3. **文档编写**：在 `docs/components` 目录下编写组件文档
4. **构建发布**：使用 `pnpm build:lib` 构建组件库

### 集成优势

- **统一标准**：确保公司所有业务系统的UI一致性
- **快速开发**：复用成熟组件，减少重复开发工作
- **维护成本低**：集中管理组件，统一升级和修复
- **技术沉淀**：积累公司业务场景的最佳实践

### 环境要求

- Node.js >= 18
- pnpm >= 8
- Vue >= 3.5.13

## 🤝 贡献指南

我们欢迎公司内部开发者积极参与组件库的建设和改进。

### 贡献流程

1. **提出需求**：在内部平台创建issue，描述组件需求
2. **设计评审**：团队评审组件设计和API
3. **开发实现**：按照组件库规范实现功能
4. **测试验证**：确保组件在不同场景下的稳定性
5. **文档编写**：完善组件文档和使用示例
6. **代码审查**：通过团队代码审查
7. **发布上线**：合并代码并发布新版本

### 开发规范

- 遵循Vue 3 Composition API最佳实践
- 使用TypeScript进行类型定义
- 组件命名采用PascalCase格式
- 样式使用Less预处理器
- 代码风格遵循ESLint规则
